Last edited time: May 21, 2026 11:49 AM
Number: 1

### `I. Login Interface`

**Function:**

- Display login screen with username and password input fields.
- Appear on dashboard startup or when user logs out.
- Require authentication before accessing main dashboard.

---

### **`II. Authentication`**

**Function:** 

- Verify username and password against Main Device database.
- Hash and compare passwords securely.
- Determine user role (Admin, Manager, User) upon successful authentication.
- Grant appropriate access level based on role.
- Deny access and display error message on failed authentication.

---

### **`III. Session Management`**

**Function:** 

- Store authentication token/session for logged-in user with their role information.
- Maintain session across page refreshes.
- Allow logout functionality to clear session and return to login screen.
- Track which user is currently logged in and their role.

---

### `IV. User Account Management (Admin Only)`

**Function:** 

- Provide interface for Admin to create, edit, and delete user accounts.
- Assign user roles (Admin, Manager, User).
- Admin can change other users' roles and passwords.
- Users can create their own account as "User" role.

---

### **`V. User Database Integration`**

**Function:** 

- Interface with Main Device database to store and retrieve user credentials and roles.
- Hash passwords securely before storage.
- Support account creation and role assignment.
